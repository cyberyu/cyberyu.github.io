readDB(conn, **kwargs)

source code 

Return the textual Google Search Results data for feature engineering in Relevance Filtering.

Parameters:
  • conn (psycopg2 connection object) - The established connection object to the Google Search Result database
  • kwargs - remaining keyword arguments are passed to readDB functon, see below

    Args:

  • textfield (list object) - the list of Google Search Result table names to be concatenated as text field. Example: ['fulltext', 'srtitle']
  • training_srids (list object) - the list of document ids used as training data
  • test_srids (list object) - the list of document ids used as test data
Returns:
A python dictionary {"docs": docs, "labels": label, "srids":srids}. Docs are a list of text. Labels are a list of relevance flags used for training. srids is a list of document ids.

updateDB(conn, **kwargs)

source code 

Update the Google Search Results table after the test data has been predicted with relevance labels and relevance scores.

Parameters:
  • conn (psycopg2 connection object) - The established connection object to the Google Search Result database
  • kwargs - remaining keyword arguments are passed to readDB functon, see below

    Args:

  • idlist (list object) - the list of document ids whoses labels or relevance scores need to be updated.
  • type (string object) - the type of values to be updated. If type='label', then update the relevance flags of the documents. If type='relevanceScore', then update the relevance scores of the documents
  • value (list object) - the list of values need to be updated
Returns:
null
